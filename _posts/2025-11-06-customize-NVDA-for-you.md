# 스크린리더에 나만의 기능을 추가해 봅시다 - NVDA 플러그인 제작 실습



## 1\. 준비물

1. 인터넷이 연결된 컴퓨터(MS 윈도우 운영체제)
2. 끈기



## 2\. 사전 지식

1. 웹브라우저에서 홈페이지를 검색하고 파일을 다운로드할 수 있습니다.
2. 응용프로그램을 설치할 수 있습니다.
3. 명령프롬프트(cmd.exe)가 무엇인지 알고 있습니다.
4. 파이썬 언어로 간단한 프로그램을 만들고 실행할 수 있습니다.



## 4\. 오픈소스 스크린리더 NVDA 설치

2025년 11월 8일 현재 최신 NVDA 버전은 2025.3.1입니다.
[여기](https://www.nvaccess.org/download/)에서 다운로드하고 설치하세요.

## 5\. NVDA 소스 코드 내려 받기

### 5.1. git 설치

git 프로그램을 사용하면 NVDA 소스코드를 내려 받아 볼 수 있습니다.
2025년 11월 8일 현재 최신 git 프로그램은 [여기](https://github.com/git-for-windows/git/releases/download/v2.51.2.windows.1/Git-2.51.2-64-bit.exe)에서 내려 받아 설치하세요.

### 5.2. git으로 소스 내려 받기

먼저 아래와 같이 명령프롬프트(cmd.esxe)를 실행합니다.

1. 윈도우 키를 누르면 시작 메뉴가 열립니다.
2. 검색 상자에서 'git cmd'를 입력하고 팝업 키를 누릅니다.
3. 팝업 메뉴에서 관리자 권한을 선택하세요.

다음으로 소스코드를 내려 받을 폴더로 이동합니다.
만약 D드라이브 루트에 내려 받고 싶다면 다음과 같이 하세요.

d:<엔터키>

다음 명령을 입력하고 엔터키를 누르면 D드라이브 루트에 nvda 폴더가 생겨나고 거기에 NVDA 소스 코드가 저장됩니다.

git clone --recursive -b release-2025.3.1 https://github.com/nvaccess/nvda

소스 코드를 보고 싶다면 윈도우 탐색기에서 D드라이브에 nvda 폴더로 이동해 보세요.

### 5.3. NVDA를 직접 수정하고 싶어요.

만약 파이썬 언어에 능숙하다면 플러그인이 아니라 NVDA 자체를 직접 수정하고 빌드할 수 있습니다.
다음 폴더를 참고하여 개발 환경을 구축하고 직접 빌드해 보세요.

createDevEnvironment.md

이 파일에는 개발 환경 구축 방법이 나와 있습니다.
이문서는 영어지만 인공지능으로 번역해 보시면 그리 어렵지 않을 겁니다.
번역한 내용이 어렵다면 NVDA 직접 빌드는 다음 기회로 미루세요.

buildingNVDA.md

이 파일에는 NVDA 빌드의 예시가 나와 있습니다.

## 6\. 플러그인이란?

NVDA에서 플러그인이란 기본 기능 이외의 추가 기능을 의미합니다.
추가 기능 개발에 대한 대략적인 내용은 [여기](https://download.nvaccess.org/documentation/developerGuide.html)에서 확인하세요.

우리가 주목할 플러그인은 앱모듈과 글로벌 플러그인 두 종류입니다.

앱모듈은 특정한 애플리케이션에 대응하는 플러그인입니다.
예를 들면 메모장에서만 동작하는 기능을 추가하고 싶다면 앱모듈을 만드세요.

글로벌 플러그인은 윈도우의 모든 곳에서 사용할 수 있는 기능입니다.



## 7\. scratchpad

NVDA에는 플러그인을 개발하고 테스트할 수 있는 특수한 폴더를 가지고 있습니다.
이 폴더의 이름이 바로 scratchpad입니다.
아래와 같이 활성화하세요.

1. insert+n: NVDA 메뉴 열기
2. 설정 -> NVDA 설정 <엔터>
3. 고급 설정<탭>
4. 이 설정을 변경하면 NVDA가 비정상적으로 동작할 수 있다는 점을 이해했습니다. 체크상자 <스페이스>로 체크
5. Developer Scratchpad 디렉토리에서 코드를 불러들일 수 있습니다 체크상자에서 <스페이스>로 체크
6. <엔터>로 설정 저장

NVDA를 재시작하면 scratchpad 폴더가 활성화 됩니다.

scratchpad 폴더는 다음과 같이 열 수 있습니다.

1. 파일 탐색기를 실행합니다.
2. Alt+D를 누릅니다.
3. %appdata%<엔터>
4. nvda 폴더에 들어가면 scratchpad를 찾을 수 있습니다.

scratchpad 폴더에는 다음과 같이 주목할 두 개의 폴더가 있습니다.

1. appModules: 앱모듈을 테스트하는 곳
2. globalPlugins: 글로벌 플러그인을 테스트하는 곳.
